## Changes Made


- ✅ `getSavedReports()` - Fetch saved reports with filters & pagination

- ✅ `downloadSavedReportCSV()` - Convert any report to CSV and download

**Download Function Features:**
- Handles multiple report types (rows, payments, portfolios, sessions)
- Proper CSV escaping for special characters
- Automatic filename generation with date
- Generic fallback for unknown report structures

## How To Use

### For Superadmins

1. **Navigate to Reports Page**
   ```
   Dashboard → Reports
   ```

2. **Click "Saved Reports (Database)" Button**
   - See all historically saved reports
   - Filter by type or date range

3. **Download Any Report**
   - Click the "Download CSV" button in the Actions column
   - File downloads automatically
   - Filename format: `{reportType}_{id}_{date}.csv`

4. **Browse Pages**
   - Use pagination controls at bottom
   - 20 reports per page (adjustable in code)

### Example Usage Flow

```
1. Generate a report (e.g., Batch Attendance)
   → Report is AUTO-SAVED to database

2. Later, go to "Saved Reports" section
   → See the report in the list
   
3. Click "Download CSV"
   → Get CSV file with all data
```

## Technical Details

### Database Schema
```sql
CREATE TABLE reports (
  id INT AUTO_INCREMENT PRIMARY KEY,
  reportType VARCHAR(100),
  reportName VARCHAR(255),
  generatedBy INT,
  parameters JSON,
  data JSON,              -- ← Complete report data stored here
  summary JSON,
  recordCount INT,
  status ENUM(...),
  createdAt DATETIME,
  ...
);
```

### Data Flow
```
User generates report 
  ↓
Backend saves to DB (with full data payload)
  ↓
Frontend fetches via /api/reports/saved
  ↓
Displays in table with other saved reports
  ↓
User clicks download
  ↓
Backend converts JSON data → CSV
  ↓
Browser downloads file
```

### CSV Generation Logic
The download function intelligently handles different report structures:

1. **Array-based reports** (rows, payments, portfolios)
   - Extracts headers from first object
   - Converts each row to CSV line

2. **Session-based reports** (batch attendance)
   - Nested loop through sessions and attendances
   - Flattens structure for CSV

3. **Other reports** (generic)
   - Key-value pairs format
   - Stringifies complex objects
